## 0. Product concept (do not abandon)
 
AntiDuplPlus is a **Windows x64** desktop app that:
 
1. Builds portable image databases via **NvJpegCollector** (GPU JPEG decode).  
2. Manages multiple DBs and **Pool1/Pool2** modes.  
3. Runs **GPU AllVsAll** (Squared Sum / SSIM) for near-duplicate search.  
4. Lets users delete/move/auto-select results; data lives next to the exe.
 
Do **not** rewrite as a cloud service, drop collector format, or replace WinForms as primary UI unless the user explicitly changes product direction.

## 3. Design principles for this track
 
1. **Safety before features** — bounds checks on every load/compare buffer.  
2. **One source of truth for thumb size** — registry `ThumbSize` == options `reducedImageSize` for a search, or refuse.  
3. **GPU is an accelerator, not a different product** — same filters as CPU unless UI documents otherwise.  
4. **Core owns file identity** — UI must not `File.Move` without updating result/image storage.  
5. **Default preserves behavior** — new settings default to current behavior.  
6. **Fail loud** on corrupt DB; fail open only on optional niceties.  
7. **Karpathy** — minimal patches; no framework rewrites.

## 8. Ready-to-paste micro-patches (highest ROI)
 
### A. Magic (WP-1)
 
```cpp
// Load() format detect
isDllNative = (firstBytes == 0x69696461u); // "adii"
```
 
### B. Thumb byte check (WP-0)
 
```cpp
size_t expected = (size_t)fileThumbSize * (size_t)fileThumbSize;
if (thumbBytes != expected) { fclose(f); return false; }
```
 
### C. Pack side guard (WP-0)
 
```cpp
if ((size_t)pImageData->data->side != reducedImageSize)
    continue;
```
 
### D. Empty GPU success (WP-4)
 
```cpp
if (validCount < 2)
    return true;
```
 
### E. Shutdown (WP-2)
 
```csharp
startFinishForm.WaitForWorker(System.Threading.Timeout.Infinite);
```
 
(Implement `WaitForWorker` to Join without forced 2s if needed.)

## 9. Anti-patterns
 
| Do not | Why |
|--------|-----|
| Change collector `fwrite` field order “for cleanliness” | Breaks all existing DBs |
| Detect DLL-native with `"adid"` | Data magic ≠ index magic |
| `memcpy` thumb using options size without checking `data->side` | OOB |
| `File.Move` without core update | Stale results |
| Raise CUDA batch / remove pitch align “for speed” | Illegal address / slower consumer GPUs |
| Silent catch around load/save of DBs | Hides corruption |
| Drive-by WPF / rename projects | Out of track |
| “Unify” formats into one without migration tool | Data loss risk |
| Edit `adExternal.h` / `External.cs` by hand | Auto-generated from `version.txt` |

## 10. Success metrics (definition of done)
 
| Metric | Target |
|--------|--------|
| OOB on mismatched thumb | Impossible (reject/guard) |

## 13. Summary for incoming agent
 
You are improving **reliability** of an already-shipping GPU duplicate finder. Primary production path is **Collector-native DBs + GPU AllVsAll**. The highest-value work is:
 
1. **Stop buffer overruns** when thumb sizes disagree or files are corrupt (WP-0).  
2. **Fix DLL index magic** so classic cache works (WP-1).  
3. **Make move/exit consistent** with core state (WP-2).  
4. **Align GPU with user compare options** (WP-3).  
5. Then harden GPU error paths (WP-4).
 
Do not redesign the product. Do not change the collector on-disk format without a versioned migration. Prefer small patches with manual smoke over large refactors.
 
**Suggested first command:**  
`implement WP-0 only` (BUG-01 + BUG-03 from the full audit).
